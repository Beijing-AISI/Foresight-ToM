# Defense Dataset

This folder contains the dataset used by the defense experiments in `../main_defense.py`.

## Files

### `defense_samples_with_true_args.json` (344 KB)

**Main dataset — 120 samples, 12 configs × 10 samples each**

- **Source**: `Attack/results/attack_experiment_summary.json` + Hi-ToM data files
- **Victim model** (that the attack was originally run against): `llama3.1:8b`
- **Attacker model** (that generated `false_argument`): `qwen3.5:9b`
- **Selection**: samples where the victim's initial answer was correct and an attack was launched, prioritizing samples where the attack actually flipped the answer; 10 samples kept per config
- All model-response fields from the original attack run (`init_letter`, `init_text`, `init_correct`, `final_letter`, `final_text`, `final_correct`, `flipped`, `elapsed_s`) were stripped so the dataset is model-agnostic — only the task input and the attack design remain
- `true_argument` was added afterward: a `glm4:9b`-generated argument, length-matched (~500 chars) to `false_argument`, that argues honestly for the correct answer instead of the attack target. This lets every scenario be tested against both a misleading and a truthful argument (see `../RESULTS.md`).

**Configs** (12, 10 samples each):
```
No_Tell_CoT_len1, No_Tell_CoT_len2, No_Tell_CoT_len3
No_Tell_MC_len1,  No_Tell_MC_len2,  No_Tell_MC_len3
Tell_CoT_len1,    Tell_CoT_len2,    Tell_CoT_len3
Tell_MC_len1,     Tell_MC_len2,     Tell_MC_len3
```
(`Tell` vs `No_Tell` = whether the story explicitly states a character left the room; `CoT` vs `MC` = chain-of-thought vs multiple-choice question style; `len1/2/3` = story length tier.)

**Structure**:
```json
{
  "metadata": {
    "source": "Attack/results/attack_experiment_summary.json + Hi-ToM data files",
    "victim_model": "llama3.1:8b",
    "attacker_model": "qwen3.5:9b",
    "total_selected": 120,
    "modifications": [
      "Renamed 'attack_text' to 'false_argument'",
      "Added 'true_argument' field (generated by glm4:9b)"
    ]
  },
  "configs": {
    "config_name": {
      "config_info": {...},
      "total_available": 0,
      "selected_count": 10,
      "samples": [
        {
          "sample_id": 6,
          "order": 0,
          "tell": "No_Tell",
          "prompt": "CoT",
          "length": 1,
          "story": "...",
          "question": "Where is the apple really?",
          "choices": {"A": "...", "...": "..."},
          "correct_answer": "red_box",
          "target_letter": "O",
          "target_text": "blue_cupboard",
          "false_argument": "Arguing for: [O]. blue_cupboard\n...",
          "true_argument": "Arguing for: [N]. red_box\n..."
        }
      ]
    }
  }
}
```

**Fields:**

| Field | Meaning |
|---|---|
| `sample_id`, `order`, `tell`, `prompt`, `length` | Config metadata. `order`: 0=reality, 1=1st-order belief, 2=2nd-order belief |
| `story`, `question`, `choices`, `correct_answer` | The task itself |
| `target_letter`, `target_text` | What the attack argues for — the wrong answer a successful attack would push the model toward |
| `false_argument` | The misleading argument (from the original attack run) that argues for `target_text` |
| `true_argument` | An honest, length-matched argument (generated later) that argues for `correct_answer` instead |

`false_argument`/`true_argument` are what `main_defense.py --argument-type {false,true}` injects into the `[ARGUMENT]` section of each scenario's prompt.

### `selection_summary.json` (1.2 KB)

Per-config selection stats: how many of the 10 selected samples per config were originally flipped by the attack (`flipped`) vs. held their correct answer (`held`). 120 total: 110 flipped / 10 held.

## Original data lineage

- Original pool: 720 samples (12 configs × 3 orders × 20 samples)
- After filtering to `init_correct=True`: 315 available
- This dataset: 120 selected (10 per config, prioritizing flipped samples)
